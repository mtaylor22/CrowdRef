Documentation generation configuration file

Installation

  • Edit amt_keys.php to specify your organization's access keys for the AMT service.
  • Copy amt_rest_api.php and amt_keys.php to your include directory.
  • If you intend to use the notification API, copy amt_notification.php to your include directory.
  • If you intend to use qualifications, copy amt_qual.php to your include directory.
  • If you intend to use configuration files to define HIT types, copy amt_config.php to your include directory.
  • If you intend to dump results in CSV format, copy csvwriter.php to your include directory.

Documentation

The class documentation is best generated using the docblox system. Under Linux, the recommended procedure is to do the following from the command-line:

$ cd ~/path_to_amt_install_dir

$ sudo pear install docblox

$ docblox run

This will generate the package documentation in the doc subdirectory. There are bidirectional links to doc/manual.html.
